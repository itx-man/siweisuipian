大模型时代，如何高效学习新技术

在 LLM 触手可及的今天，学习新技术的范式已悄然转变。我们不再需要死记 API 文档，而是学会提问的艺术——用精准的问题换取深刻的解释。

旧范式的崩塌

过去的技术学习路径几乎是固定的：找一本书、刷教程、做练习、踩坑、爬出来。这套方法有效，但效率极低。大量时间花在了「查」上——查文档、查 Stack Overflow、查 GitHub Issues。

现在，这个「查」的环节已经被大幅压缩。但这也带来了一个新的问题：我们开始不求甚解了。

提问即是学习

我发现，真正让 AI 成为有效学习工具的，是把它当作一个苏格拉底式对话伙伴，而不是答案机器。

「给我解释一下 Rust 的生命周期」不如「用一个具体的例子，解释为什么编译器拒绝了这段代码，以及它在保护我免受什么问题」

后者迫使你在提问之前先思考，同时也让 AI 的回答更有针对性、更容易内化。

我的三层学习法

第一层：快速建立直觉。用 AI 快速扫描新技术的核心概念，建立初步的心智模型。这一阶段允许模糊，目标是能「感觉到」这个技术在解决什么问题。

第二层：动手验证假设。带着疑问去写代码，让错误暴露理解中的漏洞。这一阶段 AI 充当即时的调试伙伴。

第三层：向他人解释。写博客、做分享，或者直接对着 AI 复述你的理解，让它指出错误。费曼技巧在 AI 时代获得了新的载体。

警惕的陷阱

大模型最危险的地方，是它总能给出一个「听起来正确」的答案。这会让我们产生一种虚假的理解感。

解药是：坚持写代码。代码会以最直接的方式告诉你，你到底懂没懂。

# 这段代码能跑，但你真的理解为什么吗？
def memoize(func):
    cache = {}
    def wrapper(*args):
        if args not in cache:
            cache[args] = func(*args)
        return cache[args]
    return wrapper

如果你能不借助 AI，清晰地解释这段代码中闭包的工作原理、为什么 cache 不会被垃圾回收——那你才真的懂了。
